# 使用方法

xlsx-agent は、Excelファイルを自然言語の指示で編集するツールです。
ユーザーは**ブラウザからアクセスして操作するだけ**です。編集はサーバーPC側で行われ、
差分プレビューを確認・承認してからダウンロードします。元ファイルは承認するまで
変更されません。

---

## 1. サーバーPC側（最初に一度だけ）

### 必要なもの
- Python 3.10 以上
- [Ollama](https://ollama.com)

### 手順

```bash
# モデルを取得（初回のみ・約9.6GBのDL）
ollama pull gemma4-e4b:latest
ollama serve                  # 常駐していなければ起動

# リポジトリを取得して起動
git clone https://github.com/sig65536/xlsx-agent.git
cd xlsx-agent
./start.sh                    # Windowsは start.bat をダブルクリック
```

起動に成功すると、次のように待ち受けアドレスが表示されます。

```
[run] http://0.0.0.0:8000  (ブラウザからアクセス) / model=gemma4-e4b:latest
```

サーバーPCのIPアドレスを確認しておきます（例 `192.168.1.50`）。
- Linux: `ip addr` / `hostname -I`
- Windows: `ipconfig`

---

## 2. ユーザー側（ブラウザで操作）

同じネットワーク上の任意のPCから、ブラウザで次のURLを開きます。

```
http://<サーバーPCのIP>:8000
        例: http://192.168.1.50:8000
```

### 操作の流れ

1. **ファイルを選ぶ** … `.xlsx` または `.xlsm`（最大20MB）をドラッグ＆ドロップ、またはクリックして選択
2. **シート名（任意）** … 空欄ならアクティブシートが対象
3. **指示を入力** … やりたいことを日本語で入力
   - 例:「C列の売上を合計して、最終行の下に『合計』行を追加して」
   - 例:「B列が空白の行を削除して」
   - 例:「1行目を見出しとして太字にし、背景をグレーにして」
   - 例:「金額が1000未満のセルを赤字にして」
   - 例:「日付列を yyyy/mm/dd 形式に整えて」
   - 複数の作業をまとめて指示してもOK（エージェントが順に処理します）
4. **「編集を依頼」** を押す … サーバーで解析→LLMが編集→適用 が進む
5. **差分プレビューを確認** … 変更されたセルが「変更前 / 変更後」で一覧表示される
6. **「承認してダウンロード」** … 問題なければ承認。編集済みファイルが自動ダウンロードされる
   - やり直したい場合は「やり直す」で最初に戻る

> ダウンロードURLは**一度きり**有効です。元ファイルは承認するまで書き換わりません。

---

## 3. うまくいかないときは

| 画面に出るエラー | 意味 / 対処 |
|---|---|
| `LLM_MODEL_NOT_FOUND` | サーバーでモデル未取得。`ollama pull gemma4-e4b:latest` を実行 |
| `LLM_TIMEOUT` | Ollamaが起動していない／応答が遅い。`ollama serve` を確認。もう一度試す |
| `EXCEL_SAVE_VALIDATION_FAILED` | 生成結果のファイルが壊れていたため破棄（元ファイルは無事）。指示を具体的にして再実行 |
| `CODE_CHECK_FAILED` | 指示が複雑すぎて安全なコードを生成できなかった。指示を分割・簡潔に |
| `UNSUPPORTED_FORMAT` | `.xlsx` / `.xlsm` 以外、または壊れたファイル |
| `FILE_TOO_LARGE` | 20MB超。ファイルを分割する |
| ブラウザで画面が出ない | サーバーのIP・ポート(8000)、同一ネットワークか、ファイアウォール設定を確認 |

## 4. 設定（任意）

環境変数で挙動を変更できます（指定しなければ既定値で動作）。

| 変数 | 既定値 | 説明 |
|---|---|---|
| `PORT` | `8000` | 待ち受けポート |
| `HOST` | `0.0.0.0` | 待ち受けホスト |
| `OLLAMA_MODEL` | `gemma4-e4b:latest` | 使用モデル |
| `OLLAMA_ENDPOINT` | `http://127.0.0.1:11434/api/generate` | Ollama の API |
| `LLM_TIMEOUT_SECONDS` | `60` | LLM応答タイムアウト |
| `CORS_ORIGINS` | `*` | 許可するアクセス元（カンマ区切りで限定可） |

例（ポート9000で起動）:

```bash
PORT=9000 ./start.sh
```

---

## 5. 注意

- 機密ファイルを扱う場合は、必ず原本のバックアップを保持してください。
- このツールは信頼できるネットワーク内での利用を前提としています。インターネットに直接公開しないでください。
